> **সময়:** প্রায় ১৫-২০ মিনিট
> **আগের ধাপ:** [← Node.js ইনস্টল](./04-nodejs-install.md)

---

## 🎯 এই ধাপে যা শিখবেন

- কোড ডেলিভারি করার ৩টি **Industry Standard Level** সম্পর্কে জানা
- **Level 1 (Manual Standard):** SSH Deploy Keys সেটআপ করা (বিস্তারিত গাইড)
- **Level 2 & 3:** অটোমেশন ও আধুনিক স্ট্যান্ডার্ডের একটি রোডম্যাপ

---

## 🗺️ কোড ডেলিভারি করার ৩টি লেভেল

| লেভেল | নাম                 | কখন ব্যবহার করবেন                      | কেন ব্যবহার করবেন                         |
| :---- | :------------------ | :------------------------------------- | :---------------------------------------- |
| **১** | **SSH Deploy Keys** | ✅ বর্তমানে শিখছেন / ছোট প্রজেক্ট      | সবচেয়ে নিরাপদ ও স্ট্যান্ডার্ড শুরুতে      |
| **২** | **GitHub Actions**  | বড় প্রজেক্ট / অটোমেশন দরকার            | `git push` দিলেই অটো আপডেট হবে            |
| **৩** | **Docker Registry** | স্কেলেবল অ্যাপ / আধুনিক মাইক্রোসার্ভিস | এটি আধুনিক জগতের সবচেয়ে শক্তিশালী সিস্টেম |

## 🚀 ১. লেভেল ১ (মানুয়াল স্ট্যান্ডার্ড): কোড ক্লোন করার পদ্ধতি

> **এটি ইন্ডাস্ট্রি স্ট্যান্ডার্ডের প্রথম ধাপ।** এখানে আমরা পিসি থেকে কোড GitHub-এ পুশ করবো আর সার্ভার সেটি ম্যানুয়ালি নামিয়ে নিবে। নিচে দুটি অপশন দেওয়া হলো:

---

### 🟢 অপশন ১: SSH Deploy Keys (সবচেয়ে নিরাপদ ও প্রফেশনাল)

> **কেন করবেন?** এতে আপনার পাসওয়ার্ড বা টোকেন টাইপ করতে হয় না এবং এটি অত্যন্ত নিরাপদ।

#### ধাপ ১.১ — সার্ভারে চাবি (SSH Key) তৈরি করুন

```bash
ssh-keygen -t ed25519 -C "your-email@example.com"
```

_(এন্টার চাপতে থাকুন যতক্ষণ না শেষ হয়)_

#### ধাপ ১.২ — চাবিটি কপি করুন

```bash
cat ~/.ssh/id_ed25519.pub
```

_(আউটপুটে আসা গিজিগিজি টেক্সটটি পুরো কপি করুন)_

#### ধাপ ১.৩ — GitHub-এ যোগ করুন

১. GitHub Repo → **Settings** → **Deploy keys** → **Add deploy key**।
২. চাবিটি পেস্ট করুন এবং সেভ করুন।

#### ধাপ ১.৪ — SSH দিয়ে Clone করুন

```bash
cd ~/projects
git clone git@github.com:your-username/my-app.git
cd my-app
npm install
```

---

### 🟡 অপশন ২: Personal Access Token (সবচেয়ে সহজ ও দ্রুত)

> **কেন করবেন?** যদি আপনি SSH সেটআপ নিয়ে ঝামেলা করতে না চান এবং খুব দ্রুত প্রজেক্ট ক্লোন করতে চান।

#### ধাপ ২.১ — GitHub টোকেন তৈরি করুন

১. GitHub → **Settings** (একদম উপরে প্রোফাইল থেকে) → **Developer Settings**।
২. **Personal Access Tokens** → **Tokens (classic)** → **Generate new token**।
৩. প্রজেক্টের নাম দিন এবং অন্তত `repo` পারমিশন সিলেক্ট করে জেনারেট করুন। টোকেনটি কপি করে রাখুন।

#### ধাপ ২.২ — টোকেন দিয়ে Clone করুন

```bash
cd ~/projects
git clone https://YOUR_TOKEN_HERE@github.com/your-username/my-app.git
cd my-app
npm install
```

---

### 🔄 কোড আপডেট (দুই পদ্ধতির জন্যই একই)

যখনই আপনি আপনার পিসিতে কোড আপডেট করে GitHub-এ পুশ করবেন, সার্ভারে এসে শুধু এই ৩টি কমান্ড দিন:

```bash
cd ~/projects/my-app
git pull
npm install       # নতুন package থাকলে
npm run build     # নতুন build তৈরি হবে
```

> ✅ **ব্যাস! শুধু `git pull` + `npm run build`** দিলেই আপনার সাইট আপডেট হয়ে যাবে। এখন আর পাসওয়ার্ড বা টোকেন লাগবে না।

## ⚙️ ২. লেভেল ২ (অটোমেশন স্ট্যান্ডার্ড): CI/CD (GitHub Actions)

> **⚠️ এটি পূর্ণাঙ্গ গাইড নয়:** এটি একটি উন্নত পদ্ধতি যা আপনি ভবিষ্যতে শিখবেন যখন আপনার প্রজেক্ট বড় হবে।

**এটি কী?**
এখানে আপনাকে সার্ভারে ঢুকে ম্যানুয়ালি `git pull` দিতে হবে না। আপনি আপনার পিসিতে কোড পুশ করার সাথে সাথেই GitHub নিজ দায়িত্ব নিয়ে সার্ভারে কোড পাঠিয়ে দেবে এবং স্বয়ংক্রিয়ভাবে `build` ফাইল তৈরি করে অ্যাপ রিস্টার্ট করে দিবে।

**কীভাবে কাজ করে?**  
GitHub-এর ভেতরে আপনাকে একটি `.yml` ফাইল দিয়ে কোড লিখে দিতে হয়। এটি তখন একশন মোডে (Action) চলে যায়।

---

## 🐳 ৩. লেভেল ৩ (আধুনিক স্ট্যান্ডার্ড): Docker Image Registry

> **⚠️ এটি পূর্ণাঙ্গ গাইড নয়:** এটি বর্তমানে ইন্ডাস্ট্রি স্ট্যান্ডার্ডের সবচেয়ে পাওয়ারফুল ধাপ। বড় বড় কোম্পানি যেমন ফেসবুক বা গুগল এই পদ্ধতি ব্যবহার করে।

**এটি কী?**  
এখানে সার্ভারে সরাসরি কোড নিয়ে যাওয়া হয় না। বরং আপনার পুরো অ্যাপটির একটি "ইমেজ" (Image) তৈরি করা হয় এবং সেটি একটি স্টোরহাউসে (DockerHub/AWS ECR) রাখা হয়। সার্ভার শুধু সেই রেডিমেড ইমেজটি নামিয়ে নিয়ে চালায়।

**কেন এটি সেরা?**

- এটি কোনো `Local vs Server OS`-এর ঝামেলা করে না।
- এক সেকেন্ডে হাজার হাজার সার্ভারে একই অ্যাপ চালানো যায়।

## 📦 প্রোডাকশন বিল্ড তৈরি করুন

> **লেভেল ১** সফলভাবে করলে এটি আপনার সাইট চালানোর রেডি ফাইল তৈরি করবে।

```bash
cd ~/projects/my-app

```
