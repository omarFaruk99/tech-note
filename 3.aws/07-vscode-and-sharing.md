## 📂 পার্ট ০: `.ssh` ফোল্ডার কী এবং কেন দরকার?

`.ssh` হলো আপনার কম্পিউটারের **চাবির বাক্স (key locker)**। SSH দিয়ে সার্ভারে ঢোকার সময় যা লাগে — চাবি, সেটিংস, পরিচিত সার্ভারের তালিকা — সব এখানে থাকে।

| ফাইল              | কাজ                                                    |
| ----------------- | ------------------------------------------------------ |
| `config`          | কোন সার্ভারে কিভাবে কানেক্ট করবেন তার **সেটিংস**       |
| `*.pem`           | AWS থেকে পাওয়া **চাবি ফাইল**                          |
| `known_hosts`     | আগে কানেক্ট করা **পরিচিত সার্ভারের তালিকা**            |
| `authorized_keys` | (সার্ভারে থাকে) কারা ঢুকতে পারবে তার **অনুমতি তালিকা** |

> ⚠️ **সতর্কতা (Windows WSL ব্যবহারকারীদের জন্য):**
> Windows আর WSL-এর `.ssh` ফোল্ডার **সম্পূর্ণ আলাদা!**
>
> | সিস্টেম          | `.ssh` লোকেশন              |
> | ---------------- | -------------------------- |
> | **Windows**      | `C:\Users\আপনার-নাম\.ssh\` |
> | **WSL (Ubuntu)** | `/home/আপনার-নাম/.ssh/`    |
>
> VS Code-এর Remote-SSH **Windows-এর `.ssh`** ব্যবহার করে। তাই `.pem` ফাইল এবং `config` ফাইল **Windows-এর `.ssh` ফোল্ডারে** থাকতে হবে। শুধু WSL-এ রাখলে VS Code খুঁজে পাবে না!

## 🔑 পার্ট ১: `.pem` ফাইল সেটআপ (অবশ্যই করতে হবে)

সার্ভারে ঢোকার আগে আপনাকে আপনার কম্পিউটারের টার্মিনালে চাবিটি সেটআপ করতে হবে। যেহেতু আপনি অফিস এবং বাসা দুই জায়গায় কাজ করেন, তাই আপনার জন্য দুটি আলাদা নিয়ম আছে।

### দৃশ্যপট এ: অফিস (Windows WSL ব্যবহার করলে)

আপনার অফিসের পিসিতে যদি Windows থাকে এবং আপনি **WSL (Ubuntu)** টার্মিনাল ব্যবহার করেন:

```bash
# ১. উবুন্টুর ভেতরে .ssh ফোল্ডার তৈরি করুন
mkdir -p ~/.ssh

# ২. Windows-এর Downloads থেকে ফাইলটি উবুন্টুতে নিয়ে আসুন
# (আপনার পিসিতে পাথ হলো: /mnt/host/c/Users/saaqe/Downloads/)
cp /mnt/host/c/Users/saaqe/Downloads/my-react-server-key.pem ~/.ssh/

# ৩. ফাইলের পারমিশন ঠিক করুন
chmod 400 ~/.ssh/my-react-server-key.pem
```

> 🔴 **VS Code ব্যবহার করলে অতিরিক্ত ধাপ:**
> VS Code Windows-এর SSH ব্যবহার করে, তাই `.pem` ফাইলটি **Windows-এর `.ssh` ফোল্ডারেও** কপি করতে হবে:
>
> ```powershell
> # PowerShell বা CMD দিয়ে চালান:
> copy C:\Users\saaqe\Downloads\my-react-server-key.pem C:\Users\saaqe\.ssh\
> ```

### দৃশ্যপট বি: বাসা (সরাসরি Ubuntu ব্যবহার করলে)

আপনার বাসার পিসিতে যদি সরাসরি **Ubuntu OS** থাকে:

```bash
# ১. .ssh ফোল্ডার তৈরি করুন
mkdir -p ~/.ssh

# ২. সরাসরি Downloads ফোল্ডার থেকে ফাইলটি কপি করুন
cp ~/Downloads/my-react-server-key.pem ~/.ssh/

# ৩. ফাইলের পারমিশন ঠিক করুন
chmod 400 ~/.ssh/my-react-server-key.pem
```

## 💻 পার্ট ২: VS Code দিয়ে কানেক্ট করুন

আপনার সব কম্পিউটারে (অফিস বা বাসা) এই নিয়মগুলো একই থাকবে:

### ২.১ — Extension ইনস্টল করুন

১. VS Code খুলুন।
২. বাম পাশের **Extensions** আইকনে ক্লিক করুন।
৩. সার্চ করুন: `Remote - SSH` এবং এটি ইনস্টল করুন।

### ২.২ — SSH Config ফাইল সেটআপ করুন

১. কিবোর্ডে `F1` বা `Ctrl + Shift + P` চাপুন।
২. লিখুন: `Remote-SSH: Open SSH Configuration File...`
৩. নিচের কোডটি সেখানে পেস্ট করুন:

```text
Host my-aws-server
    HostName YOUR_PUBLIC_IP (যেমন: 13.212.19.5)
    User ubuntu
    IdentityFile ~/.ssh/my-react-server-key.pem
```

> 💡 **Public IP ব্যবহার করুন, Private IP না।** AWS console-এ আপনার instance-এ গিয়ে **Public IPv4 address** দেখুন। Private IP শুধু AWS-এর ভেতরের নেটওয়ার্কে কাজ করে।

> **আগে থেকে কোনো Host সেটআপ থাকলে কী করবেন?**
>
> Config ফাইলে আগে থেকে যদি অন্য কোনো সার্ভারের তথ্য থাকে, তাহলে **মুছবেন না!** শুধু আগের Host ব্লকের নিচে একটি **ফাঁকা লাইন** রেখে নতুনটি যোগ করুন। যেমন:

```text
# --- আগের সার্ভার (এটি আগে থেকেই আছে, মুছবেন না) ---
Host my-office-server
    HostName 192.168.1.100
    User deploy
    IdentityFile ~/.ssh/office-key.pem

# --- নতুন AWS সার্ভার (এটি নিচে যোগ করুন) ---
Host my-aws-server
    HostName 13.212.19.5
    User ubuntu
    IdentityFile ~/.ssh/my-react-server-key.pem
```

> **মনে রাখবেন:** প্রতিটি `Host` ব্লককে একটি **ফাঁকা লাইন** দিয়ে আলাদা করুন। এভাবে আপনি চাইলে ১০টি সার্ভারও একই config ফাইলে রাখতে পারবেন।

### ২.৩ — কানেক্ট করুন

১. কিবোর্ডে `F1` বা `Ctrl + Shift + P` চাপুন।
২. লিখুন: `Remote-SSH: Connect to Host...`
৩. লিস্ট থেকে **my-aws-server** সিলেক্ট করুন।

> ⚡ **প্রথমবার কানেক্ট করলে** এই মেসেজ আসবে:
> _"The authenticity of host '13.212.19.5' can't be established."_
>
> এটা স্বাভাবিক! এটা SSH-এর নিরাপত্তা প্রশ্ন — "এই সার্ভারকে চিনি না, তুমি কি নিশ্চিত?"
> শুধু **`yes`** লিখে **Enter** চাপুন। পরের বার থেকে আর এই প্রশ্ন আসবে না।

৪. কানেক্ট হয়ে গেলে VS Code-র নিচে বাম কোণায় সবুজ রঙে **SSH: my-aws-server** দেখাবে।
৫. এখন `File > Open Folder` দিয়ে আপনার সার্ভারের যেকোনো ফোল্ডার ওপেন করতে পারবেন।

## 🤝 পার্ট ৩: অন্যদের (যেমন সিনিয়রকে) এক্সেস দিন

### পদ্ধতি ১: `.pem` ফাইল শেয়ার করা (সহজ কিন্তু কম নিরাপদ)

আপনার `my-react-server-key.pem` ফাইলটি তাকে দিতে পারেন। তিনি আপনার IP এবং এই ফাইলটি ব্যবহার করে কানেক্ট করতে পারবেন।

> ⚠️ **সতর্কতা:** `.pem` ফাইল শেয়ার করলে তিনি আপনার সার্ভারের **সম্পূর্ণ এক্সেস** পাবেন। শুধু বিশ্বস্ত মানুষকে দিন। পরে এক্সেস বাতিল করতে হলে পুরো Key Pair বদলাতে হবে।

### পদ্ধতি ২: তার নিজের SSH Key যুক্ত করা (নিরাপদ পদ্ধতি ✅)

**তাকে বলুন (তার পিসিতে করতে হবে):**

```bash
# তার পিসিতে SSH Key তৈরি করতে বলুন (যদি আগে না থাকে)
ssh-keygen -t ed25519

# এরপর তার Public Key দেখতে বলুন এবং আপনাকে পাঠাতে বলুন
cat ~/.ssh/id_ed25519.pub
```

**আপনি করুন (আপনার সার্ভারে):**

```bash
# সার্ভারে ঢুকে authorized_keys ফাইল খুলুন
nano ~/.ssh/authorized_keys

# ফাইলের একদম নীচে তার Public Key পেস্ট করে সেভ করুন (Ctrl+O → Enter → Ctrl+X)
```

এরপর তিনি তার পিসি থেকে পাসওয়ার্ড ছাড়াই কানেক্ট করতে পারবেন:

```bash
ssh ubuntu@আপনার-PUBLIC-IP
```

> 💡 **সুবিধা:** পরে যদি তার এক্সেস বাতিল করতে চান, শুধু `authorized_keys` থেকে তার লাইনটি মুছে দিলেই হবে।
