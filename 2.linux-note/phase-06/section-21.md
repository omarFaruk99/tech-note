# 📖 Section 21: Environment Variables ও `~/.bashrc` কাস্টমাইজেশন

## 🎯 এই Section-এ যা শিখবেন

- Environment Variable কী এবং কেন এগুলো সিস্টেমের জন্য এত গুরুত্বপূর্ণ?
- গুরুত্বপূর্ণ সিস্টেম Variables (`$PATH`, `$USER`, `$HOME`) সম্পর্কে জানা
- সাময়িক (Temporary) এবং স্থায়ী (Permanent) Variable তৈরি করা
- `~/.bashrc` ফাইল কী এবং এটি কিভাবে কাজ করে?
- টার্মিনালের চেহারা সুন্দর করা (Custom Prompt)
- Alias (শর্টকাট কমান্ড) তৈরি করে কাজের গতি বাড়ানো

## 🌍 Environment Variables কী?

**Environment Variables** হলো এমন কিছু ডাইনামিক (পরিবর্তনশীল) মান যা অপারেটিং সিস্টেম এবং চলমান প্রোগ্রামগুলোর আচরণ নিয়ন্ত্রণ করে। সহজ ভাষায়, এগুলো হলো সিস্টেমের "গ্লোবাল সেটিংস" যা যেকোনো প্রোগ্রাম প্রয়োজনে পড়ে নিতে পারে।

### কেন দরকার?

ধরুন, আপনি একটি প্রোগ্রাম চালাচ্ছেন। সেই প্রোগ্রামটি জানতে চায়:

- আপনি কে? (এটি `$USER` নামক variable-এ লেখা থাকে)
- আপনার হোম ফোল্ডার কোথায়? (এটি `$HOME`-এ লেখা থাকে)
- আপনি কী ভাষা ব্যবহার করছেন? (এটি `$LANG`-এ লেখা থাকে)
  প্রোগ্রামটি সরাসরি লিনাক্সকে না জিজ্ঞেস করে এই ভেরিয়েবলগুলো পড়ে নেয়।

### কিছু গুরুত্বপূর্ণ সিস্টেম Variables:

| Variable | অর্থ / এর ভেতরে যা থাকে                                  |
| -------- | -------------------------------------------------------- |
| `PATH`   | লিনাক্স কমান্ডগুলো কোথায় খুঁজবে তার ফোল্ডারের তালিকা    |
| `USER`   | আপনার ইউজারনেম (যেমন: `the-king`)                        |
| `HOME`   | আপনার হোম ডিরেক্টরি (যেমন: `/home/the-king`)             |
| `SHELL`  | আপনি কোন টার্মিনাল সেল ব্যবহার করছেন (যেমন: `/bin/bash`) |
| `LANG`   | আপনার সিস্টেমের ভাষা (যেমন: `en_US.UTF-8`)               |

> 💡 **নিয়ম:** সিস্টেম Environment Variable-এর নাম সবসময় **বড় হাতের অক্ষরে (UPPERCASE)** লেখা হয়। মান দেখতে নামের আগে `$` চিহ্ন বসাতে হয়।

```bash
# যেকোনো variable-এর মান দেখতে (echo)
echo $HOME
echo $USER
```

## 1️⃣ Variable তৈরি করা এবং দেখা

### সব Variable একসাথে দেখা:

```bash
printenv
# অথবা
env
```

### সাময়িক (Temporary) Variable তৈরি:

আপনি যদি শুধু ভেরিয়েবল লেখেন, তবে তা শুধু বর্তমান টার্মিনালের জন্যই থাকবে। টার্মিনাল বন্ধ করলে মুছে যাবে।

```bash
# Local variable তৈরি (space দেওয়া যাবে না)
MY_SECRET="linux_is_awesome"

# আউটপুট দেখুন
echo $MY_SECRET

# কিন্তু অন্য কোনো স্ক্রিপ্ট বা প্রোগ্রাম এটি পাবে না।
```

### Export করা (অন্য প্রোগ্রামকেও ব্যবহার করতে দেওয়া):

`export` কমান্ড ব্যবহার করলে ভেরিয়েবলটি "Environment"-এর অংশ হয়ে যায়।

```bash
export MY_PROJECT="/var/www/html/my_website"
cd $MY_PROJECT
```

## 2️⃣ `$PATH` — লিনাক্সের জাদুকরী ভেরিয়েবল

আপনি যখন টার্মিনালে `ls` বা `grep` লেখেন, লিনাক্স কীভাবে জানে এই প্রোগ্রামগুলো কোথায় আছে? কারণ লিনাক্স `$PATH` ভেরিয়েবলে থাকা ফোল্ডারগুলোতে খোঁজে।

```bash
# আপনার PATH দেখুন
echo $PATH
# আউটপুট এমন হতে পারে:
# /usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin
```

এখানে কলন (`:`) দিয়ে অনেকগুলো ফোল্ডারের নাম একসাথে জোড়া দেওয়া হয়েছে।

### Custom Folder-কে `$PATH`-এ যুক্ত করা:

ধরুন আপনি নিজে একটি স্ক্রিপ্ট বানিয়েছেন `~/my_scripts` ফোল্ডারে। আপনি চান যেকোনো জায়গা থেকে স্ক্রিপ্টটির নাম লিখলেই যেন তা চলে।

```bash
# আপনার ফোল্ডারটিকে PATH-এর শেষে যুক্ত করে export করুন
export PATH="$PATH:$HOME/my_scripts"
```

_(তবে টার্মিনাল বন্ধ করলে এটি মুছে যাবে। চিরস্থায়ী করতে হলে একে `~/.bashrc`-তে রাখতে হয়।)_

---

## 3️⃣ `~/.bashrc` — আপনার টার্মিনালের কন্ট্রোল রুম

**`.bashrc`** (Bash Run Commands) হলো একটি লুকানো স্ক্রিপ্ট ফাইল যা আপনার হোম ডিরেক্টরিতে থাকে `(~/.bashrc)`।
আপনি যখনই নতুন কোনো টার্মিনাল (Bash shell) খোলেন, সবার আগে স্বয়ংক্রিয়ভাবে এই ফাইলটি রান হয়!

### কেন `.bashrc` এডিট করবেন?

1. **স্থায়ী Environment Variable** তৈরি করতে (যেমন উপরের `$PATH`)।
2. **Alias** (শর্টকাট) তৈরি করার জন্য।
3. টার্মিনালের চেহারা পরিবর্তন করতে।

### ফাইলটি এডিট করা:

```bash
nano ~/.bashrc
```

ফাইলটি এডিট করে সেভ করার পর পরিবর্তন সাথে সাথে কার্যকর হয় না। নতুন করে লোড করতে হয়:

```bash
# পরিবর্তন দ্রুত কার্যকর করতে
source ~/.bashrc
```

## 4️⃣ Alias (শর্টকাট কমান্ড) তৈরি করা ⚡

বড় বড় কমান্ড বারবার লেখা বেশ কষ্টের। **Alias** দিয়ে আপনি নিজের পছন্দমতো কমান্ডের ছোট নাম দিতে পারেন।

### সাময়িক Alias:

```bash
# 'll' লিখলে আসলে 'ls -lah' কাজ করবে
alias ll='ls -lah'

# 'update' লিখলে সিস্টেম আপডেট হবে
alias update='sudo apt update && sudo apt upgrade -y'
```

### স্থায়ী Alias (`~/.bashrc` এর মাধ্যমে):

যাতে প্রতিবার টার্মিনাল খোলার সময় শর্টকাটগুলো কাজ করে, এগুলো `~/.bashrc`-এর একদম শেষে যোগ করে দিন।

```bash
nano ~/.bashrc

# ফাইলের শেষে এই লাইনগুলো যোগ করুন:
# --- My Custom Aliases ---
alias update='sudo apt update && sudo apt upgrade -y'
alias clean='sudo apt autoremove -y && sudo apt clean'
alias myip='curl ifconfig.me'
alias ports='sudo ss -tulnp'
```

সেভ করে বের হয়ে আসুন এবং রিলোড দিন: `source ~/.bashrc`।

## 5️⃣ Customization (টার্মিনালের প্রম্পট সুন্দর করা)

টার্মিনালে আপনি যে লাইনটিতে কমান্ড লেখেন, যেমন `the-king@ubuntu:~$`, একে বলা হয় **Prompt**। এটি নিয়ন্ত্রিত হয় `$PS1` `(Prompt String 1)` ভেরিয়েবল দিয়ে।

```bash
# বর্তমান Prompt-এর কোডটি দেখুন
echo $PS1
```

আপনি `~/.bashrc`-তে গিয়ে এই `$PS1` পরিবর্তন করে টার্মিনালকে রঙিন ও নিজের মতো করে সাজাতে পারেন।
_(সাধারণত এটি খুব জটিল কোড হয়, তাই ডেভেলপাররা অনলাইনে "Bash Prompt Generator" থেকে কোড বানিয়ে কপি পেস্ট করেন)_।

## 🏋️ Practice Exercise

### অনুশীলন ১: Environment Variable এক্সপ্লোর করা

```bash
# 1. আপনার সিস্টেমের ভাষা কী তা চেক করুন:
echo $LANG

# 2. আপনার Home ডিরেক্টরির পাথ দেখুন:
echo $HOME

# 3. একটি নতুন ভেরিয়েবল তৈরি করুন:
export MY_NAME="The Linux King"
echo "My title is: $MY_NAME"
```

### অনুশীলন ২: Alias তৈরি এবং `.bashrc` আপডেট

```bash
# 1. আপনার .bashrc ফাইলটি এডিট করার জন্য খুলুন:
nano ~/.bashrc

# 2. ফাইলটির একেবারে শেষে (কিবোর্ডের Down অ্যারো চেপে নিচে গিয়ে) দুটি লাইন লিখুন:
alias mem='free -h'
alias disk='df -h'

# 3. Ctrl+O চেপে সেভ করুন, Enter দিন, তারপর Ctrl+X চেপে বের হয়ে আসুন।

# 4. পরিবর্তনগুলো টার্মিনালকে জানান:
source ~/.bashrc

# 5. এবার শুধু 'mem' এবং 'disk' লিখে দেখুন কতটা সহজে র‍্যাম ও ডিস্ক চেক করা যাচ্ছে!
mem
disk
```

### অনুশীলন ৩: PATH-এর কামাল

```bash
# 1. একটি ফোল্ডার তৈরি করুন
mkdir ~/bin

# 2. সেখানে একটি সহজ স্ক্রিপ্ট বানান
echo 'echo "Hello from my custom script!"' > ~/bin/greet
chmod +x ~/bin/greet

# 3. এখন greet লিখলে কাজ করবে না, কারণ ~/bin PATH-এ নেই।
# PATH আপডেট করুন:
export PATH="$PATH:$HOME/bin"

# 4. এবার শুধু greet লিখে এন্টার দিন!
greet
```
