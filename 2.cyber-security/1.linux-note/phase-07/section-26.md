## ⚡ 1. Fast Directory Navigation (দ্রুত যাতায়াত)

লিনাক্স ইউজাররা মাউস দিয়ে ফোল্ডার খোলে না। টার্মিনালে কাজ করার সময় ফোল্ডার বদলানোটা (Navigation) হতে হবে চোখের পলকে।

### `cd` এর প্র্যাকটিক্যাল ব্যবহার:

```bash
# হোম ডিরেক্টরিতে যেতে (শুধু cd লিখলেই হয়, ~/ দরকার নেই)
cd

# ঠিক আগের (Previous) ওয়ার্কিং ডিরেক্টরিতে ফিরে যাওয়া (The Undo Button)
cd -
```

_উদাহরণ: আপনি `/var/log` এ ছিলেন, এরপর `cd ~/Documents` এ গেলেন। এখন আবার `cd -` দিলে সরাসরি `/var/log` এ ফিরে যাবেন!_

### The Folder Stack (`pushd` & `popd`):

এটি `cd` এর চেয়েও স্মার্ট। ধরুন আপনার ৩-৪টি ফোল্ডারে বারবার যেতে হয়।

```bash
# কোনো ফোল্ডারে যাওয়ার সময় সেভ করে (push) যান
pushd /var/www/html
pushd /etc/nginx/sites-available

# আপনি এখন ৩ নম্বর ফোল্ডারে আছেন। আগেরগুলোতে ফিরতে চাইলে (pop):
popd   # এটি আপনাকে /var/www/html এ নিয়ে যাবে
popd   # এটি আপনাকে আপনার মূল ফোল্ডারে নেবে
```

## 🔍 2. Finding Files Like a Pro (`find`)

কোথায় কী ফাইল রেখেছেন ভুলে গেছেন? `find` হলো টার্মিনালের সার্চ ইঞ্জিন।

### নাম দিয়ে খোঁজা:

```bash
# পুরো সিস্টেমে ( / ) 'resume.pdf' ফাইলটি খুঁজুন
sudo find / -name "resume.pdf"

# বর্তমান ফোল্ডার (.) এবং এর ভেতরে সব .jpg ছবি খুঁজুন
find . -name "*.jpg"
```

### সময় দিয়ে খোঁজা (খুবই দরকারী!):

```bash
# গত ২৪ ঘন্টায় যে ফাইলগুলো মডিফাই করা হয়েছে (যেমন ডাউনলোডেড ফাইল)
find ~/Downloads -mtime 0

# গত ১০ মিনিটের ভেতর পরিবর্তন হওয়া ফাইল
find /var/log -mmin -10
```

### সাইজ দিয়ে খোঁজা (ফুল হার্ডডিস্ক ক্লিন করতে):

```bash
# ৫০০ মেগাবাইটের চেয়ে বড় ফাইল খুঁজুন
find / -size +500M

# ঠিক ৫ কিলোবাইটের রিকশা ফাইল
find . -size 5k
```

> **💡 প্রো-টিপ (Find + Action):**
> আপনি চাইলে খোঁজার সাথে সাথেই ফাইলটি ডিলিট করতে পারেন:
> `find ~/Downloads -name "*.mp4" -delete` (ডাউনলোডের সব ভিডিও মুছে ফেলবে)।

## 🔎 3. ফাইলের _ভেতরে_ লেখা খোঁজা (`grep`)

ধরুন আপনি জানেন না কোন ফাইলের ভেতর সার্ভারের পাসওয়ার্ড বা নির্দিষ্ট কোডটি লেখা আছে।

```bash
# বর্তমান ফোল্ডারের সব ফাইলের (-r মানে recursive) ভেতর "database_user" খুঁজুন
grep -r "database_user" .

# লাইন নাম্বারসহ দেখতে (-n)
grep -rn "database_user" .
```

## 👀 4. Viewing Large Files (লগ মনিটরিং)

লিনাক্সে সার্ভারের লগ ফাইলগুলো গিগাবাইট সাইজের হতে পারে। এগুলো `cat` বা `nano` দিয়ে ওপেন করলে টার্মিনাল হ্যাং হয়ে যাবে।

### `less` — বিশাল ফাইলের স্মার্ট ভিউয়ার:

```bash
less /var/log/syslog
```

_`less` এর ভেতর `Space` চেপে পরের পেজে, `b` চেপে আগের পেজে এবং `/` টাইপ করে কোনো ওয়ার্ড সার্চ করতে পারেন। বের হতে `q` চাপুন।_

### `tail -f` — লাইভ হ্যাকার ভিউ (ডেইলি অ্যাডমিন টাস্ক):

কোনো প্রসেস রান করালে লগ ফাইলে সেকেন্ডে সেকেন্ডে নতুন লাইন যুক্ত হয়। `tail -f` সেই লাইভ ডেটাগুলো রিয়েল টাইমে দেখায়।

```bash
# সিস্টেমের লাইভ মেসেজ দেখতে (Ctrl+C দিয়ে বের হবেন)
tail -f /var/log/syslog

# শুধু শেষের ১০০ লাইন থেকে লাইভ শুরু হবে
tail -n 100 -f /var/log/apache2/error.log
```

## 🏋️ Practice Exercise (Daily Routine)

### অনুশীলন ১: Fast Navigation Test

```bash
# 1. /etc ফোল্ডারে যান
cd /etc

# 2. আপনার হোম ডিরেক্টরিতে যান
cd

# 3. এক কমান্ডে আবার /etc তে ফিরে যান
cd -
```

### অনুশীলন ২: File Searching

```bash
# 1. আপনার সিস্টেমে গত ৬০ মিনিটে কোন লগ ফাইলগুলো পরিবর্তন হয়েছে খুঁজুন
sudo find /var/log -mmin -60

# 2. আপনার Home ডিরেক্টরির ভেতর 100MB এর বেশি বড় কী ফাইল আছে দেখুন
find ~ -size +100M
```
