## 📝 Log (লগ) কী এবং কেন দরকার?

**Log** মানে হলো আপনার সিস্টেমের **ডায়েরি (Diary)**। সিস্টেমে কী কী ঘটেছে (কে লগইন করেছে, কোন সার্ভিস ক্র্যাশ করেছে, কোন এরর হয়েছে) তার সম্পূর্ণ ইতিহাস লেখা থাকে Log ফাইলে।

- **Troubleshooting (সমস্যা সমাধান):** যখন কিছু কাজ করে না, তখন প্রথমে Log চেক করতে হয়।
- **Security (নিরাপত্তা):** হ্যাকার আক্রমণ করছে কিনা বা কে ব্যর্থ লগইনের চেষ্টা করেছে, তা Log থেকেই জানা যায়।

## 1️⃣ `/var/log` ডিরেক্টরি — সকল Logs-এর বাসা

Linux-এর প্রায় সব লগ ফাইল থাকে `/var/log` ডিরেক্টরিতে।

```bash
# লগ ডিরেক্টরিতে কী কী আছে দেখুন
ls -l /var/log
```

**প্রধান কিছু লগ ফাইল:**

| ফাইলের নাম                   | কী তথ্য থাকে                                                                                       |
| ---------------------------- | -------------------------------------------------------------------------------------------------- |
| `syslog`                     | সম্পূর্ণ সিস্টেমের সাধারণ লগ। যেকোনো সমস্যার জন্য এটি প্রথমে দেখা উচিত।                            |
| `auth.log`                   | **অত্যন্ত গুরুত্বপূর্ণ!** কে কখন লগইন করেছে (সফল/ব্যর্থ), `sudo` কমান্ড কে ব্যবহার করেছে তার তথ্য। |
| `dmesg`                      | কার্নেল লগের রিডাবল ভার্সন (Hardware বা Driver সংক্রান্ত তথ্য)।                                    |
| `kern.log`                   | মূল কার্নেল লগ (সরাসরি ফাইল)।                                                                      |
| `apt/...`                    | কোন সফটওয়্যার কখন ইন্সটল/আপডেট হয়েছে তার হিসেব।                                                  |
| `nginx/...` বা `apache2/...` | ওয়েব সার্ভারের এক্সেস এবং এরর লগ (যদি ইন্সটল থাকে)।                                               |

## 2️⃣ লগ ফাইল পড়ার সেরা পদ্ধতি

সাধারণত লগ ফাইল অনেক বড় হয়। তাই পুরো ফাইল `cat` দিয়ে পড়া উচিত নয়। এর বদলে অন্য টুল ব্যবহার করা ভালো।

### `tail -f` (রিয়েল-টাইমে লগ দেখা)

সবচেয়ে বেশি ব্যবহৃত কমান্ড। এটি ফাইলের শেষ ১০ লাইন দেখায় এবং **ফাইল আপডেট হওয়ার সাথে সাথে আউটপুট আপডেট করে (Follow mode)**।

```bash
# সিস্টেম লগ রিয়েল-টাইমে দেখুন (Ctrl+C দিয়ে বের হতে হবে)
sudo tail -f /var/log/syslog

# auth.log-এর শেষ ২০ লাইন এবং রিয়েল-টাইমে
sudo tail -f -n 20 /var/log/auth.log
```

### `less` (স্ক্রল করে পড়ার জন্য)

যখন পুরোনো কোনো এরর খুঁজতে হয়, তখন `less` খুব উপকারি।

```bash
sudo less /var/log/syslog
# Shift+G দিয়ে একবারে ফাইলের শেষে যাওয়া যায়
# /আপনার_শব্দ টাইপ করে খোঁজা যায়
# q চেপে বের হন
```

## 3️⃣ `grep` ব্যবহার করে লগ থেকে তথ্য খোঁজা

Section 14-এ আপনি `grep` শিখেছেন। লগ ফাইল থেকে নির্দিষ্ট তথ্য ফিল্টার করতে এর জুরি নেই।

### উদাহরণ ১: সিকিউরিটি এনালাইসিস (`auth.log`)

```bash
# কে কে ব্যর্থ (Failed) পাসওয়ার্ড দিয়েছে?
sudo grep "Failed password" /var/log/auth.log

# root ইউজার হিসেবে কে 'sudo' চালিয়েছে?
sudo grep "sudo" /var/log/auth.log

# শুধু আজকের তারিখের লগ খোঁজা (যেমন: "Mar  5")
sudo grep "Mar  5" /var/log/auth.log
```

### উদাহরণ ২: হার্ডওয়্যার সমস্যা (`dmesg` বা `syslog`)

```bash
# কোনো ড্রাইভার বা হার্ডওয়্যার "error" দিয়েছে কি না?
sudo `dmesg` | grep -i error

# USB ডিভাইস কখন লাগানো বা খোলা হয়েছে?
sudo `dmesg` | grep -i usb
```

## 4️⃣ `journalctl` — আধুনিক লগিং সিস্টেম (systemd)

Ubuntu 24-এ `journalctl` ব্যবহার করা লগ দেখার সবচেয়ে শক্তিশালী পদ্ধতি (Section 11-এও এটা নিয়ে অল্প আলোচনা হয়েছে)। এটি `/var/log` এর ফাইলগুলোর চেয়ে অনেক দ্রুত এবং ফিল্টার করতে সুবিধা।

```bash
# শুধু আজকের লগ দেখুন
journalctl --since today

# নির্দিষ্ট কোনো সার্ভিসের (যেমন ssh) লগ
journalctl -u ssh

# রিয়েল-টাইমে (tail -f এর মতো)
journalctl -f

# শুধু Error এবং উপরের লেভেলের (Critical/Alert) লগ
journalctl -p err -b
```

## 5️⃣ Log Rotation — লগ যেন খালি না করে দেয় আপনার স্টোরেজ

লক্ষ্য করেছেন কি `/var/log` এ `syslog.1`, `syslog.2.gz` এমন নামের ফাইল আছে?

**Logrotate** নামক একটি সার্ভিস প্রতিদিন/সপ্তাহে পুরনো লগগুলোকে জিপ (zip) করে রেখে দেয় এবং নতুন একটি ফাঁকা `syslog` ফাইল বানায়, যাতে আপনার হার্ডডিস্ক ভরে না যায়।

| কাজ                          | কমান্ড                                 |
| ---------------------------- | -------------------------------------- | ----------- |
| ডিরেক্টরি দেখা               | `ls -la /var/log`                      |
| রিয়েল-টাইমে শেষ লগ দেখা     | `sudo tail -f /var/log/syslog`         |

| হার্ডওয়্যার/কার্নেল লগ দেখা | `dmesg                                 | head -n 20` |
| আধুনিক systemd লগ            | `journalctl -u <সেবা> -f`              |

# 1. আপনার কার্নেলের প্রথম 15টি মেসেজ (বুট প্রসেস) দেখুন:
dmesg | head -n 15
