## ⏰ Cron কী?

**Cron** হলো Linux-এর একটি **টাইম-বেসড জব শিডিউলার**। এটি ব্যাকগ্রাউন্ডে (`cron` daemon) চলতে থাকে এবং আপনার বলে দেওয়া নির্দিষ্ট সময়ে কোনো কমান্ড বা স্ক্রিপ্ট স্বয়ংক্রিয়ভাবে চালায়।

### সহজ উদাহরণ:

| কাজ               | আপনি কিভাবে করেন                          | Cron কিভাবে করে                                   |
| ----------------- | ----------------------------------------- | ------------------------------------------------- |
| **সিস্টেম আপডেট** | প্রতিদিন সকালে টার্মিনাল খুলে কমান্ড দেন  | Cron নিজে নিজে প্রতিদিন সকাল ৬টায় কমান্ড চালায়  |
| **ডেটা ব্যাকআপ**  | প্রতি রাতে ফাইল কপি করে হার্ডডিস্কে রাখেন | Cron রাত ২টায় নিজে থেকে ব্যাকআপ স্ক্রিপ্ট চালায় |
| **লগ মুছা**       | মাস শেষে ফোল্ডার পরিষ্কার করেন            | Cron প্রতি মাসের ১ তারিখে লগ মুছে দেয়            |

## 1️⃣ `crontab` — Cron Job সেট করা

আপনার সব cron job একটি ফাইলে সেভ থাকে। এই ফাইলটি edit করতে `crontab` কমান্ড ব্যবহার করা হয়।

### Crontab কমান্ড:

```bash
# নিজের cron job ফাইল খুলুন (Edit)
crontab -e

# আপনার সেট করা সব cron job দেখুন (List)
crontab -l

# আপনার সব cron job মুছে ফেলুন (Remove - সাবধানে!)
crontab -r

# অন্য user-এর cron job edit করুন (root হিসেবে)
sudo crontab -u username -e
```

> 💡 **প্রথমবার `crontab -e` দিলে:** এটি আপনাকে একটি Text Editor বেছে নিতে বলবে। `1` চেপে **nano** নির্বাচন করুন।

## 2️⃣ Cron Job-এর Syntax (প্যাটার্ন)

একটি cron job লাইনে **৬টি অংশ** থাকে — প্রথম ৫টি সময় বোঝায়, শেষ অংশটি হলো কমান্ড।

```
* * * * * কমান্ড বা স্ক্রিপ্টের পথ
│ │ │ │ │
│ │ │ │ └─── সপ্তাহের দিন (০-৭) (রবিবার = ০ বা ৭)
│ │ │ └───── মাস (১-১২)
│ │ └─────── মাসের দিন (১-৩১)
│ └───────── ঘন্টা (০-২৩)
└─────────── মিনিট (০-৫৯)
```

> 🧠 **মনে রাখার সহজ উপায়:** "মিনিট, ঘন্টা, দিন, মাস, সপ্তাহ" (ছোট থেকে বড়)।

### বিশেষ চিহ্ন (Wildcards):

| চিহ্ন | অর্থ             | উদাহরণ                      |
| ----- | ---------------- | --------------------------- |
| `*`   | **যেকোনো** (All) | `*` (সব মাস, সব দিন)        |
| `,`   | **কমা** (তালিকা) | `1,15` (১ ও ১৫ তারিখে)      |
| `-`   | **রেঞ্জ** (থেকে) | `1-5` (সোম থেকে শুক্র)      |
| `/`   | **বিরতি** (Step) | `*/5` (প্রতি ৫ মিনিট পর পর) |

## 3️⃣ Cron Job-এর উদাহরণ

### ক. মিনিট ও ঘন্টা:

```bash
# প্রতিদিন সকাল ৬টা ৩০ মিনিটে (৩০ মিনিট, ৬ ঘন্টা)
30 6 * * * /home/user/backup.sh

# প্রতিদিন রাত ১১টা ৪৫ মিনিটে
45 23 * * * sudo apt update -y

# প্রতি ঘন্টার ১ম মিনিটে
1 * * * * echo "নতুন ঘন্টা শুরু!"

# প্রতিদিন দুপুর ১২টায়
0 12 * * * /script/report.sh
```

### খ. দিন, মাস, সপ্তাহ:

```bash
# প্রতি মাসের ১৫ তারিখ সকাল ১০টায়
0 10 15 * * /script/pay.sh

# প্রতি সোমবার সকাল ৯টায় (১ = সোমবার)
0 9 * * 1 /script/weekly-clean.sh

# শনি ও রবিবার রাত ২টায় (৬,৭ = শনি, রবি)
0 2 * * 6,7 /script/weekend.sh
```

### গ. বিরতি (Step):

```bash
# প্রতি ৫ মিনিট পর পর (সারাদিন)
*/5 * * * * /script/monitor.sh

# প্রতিদিন সকাল ৯টা থেকে বিকাল ৫টা পর্যন্ত প্রতি ২ ঘন্টা পর পর
0 9-17/2 * * * echo "কাজের সময়"
```

### ঘ. বিশেষ শর্টকাট (Macros):

কিছু ক্ষেত্রে সংখ্যা না লিখে শর্টকাট ব্যবহার করা যায়:

| শর্টকাট    | অর্থ                           | সমান Syntax |
| ---------- | ------------------------------ | ----------- |
| `@reboot`  | কম্পিউটার চালু হওয়ার সময়     | —           |
| `@hourly`  | প্রতি ঘন্টার শুরুতে            | `0 * * * *` |
| `@daily`   | প্রতিদিন মধ্যরাতে              | `0 0 * * *` |
| `@weekly`  | প্রতি সপ্তাহের শুরুতে (রবিবার) | `0 0 * * 0` |
| `@monthly` | প্রতি মাসের ১ তারিখে           | `0 0 1 * *` |
| `@yearly`  | প্রতি বছরের ১ জানুয়ারি        | `0 0 1 1 *` |

**উদাহরণ:**

```bash
# কম্পিউটার চালু হলেই স্ক্রিপ্ট চালাবে
@reboot /home/user/startup.sh

# প্রতিদিন রাতে ব্যাকআপ নেবে
@daily /home/user/backup.sh
```

## 4️⃣ Cron Job-এর Best Practices

### ১. नेहमी Absolute Path ব্যবহার করুন!

Cron-এর Environment Variable আপনার টার্মিনালের মতো নয়। তাই কমান্ড বা স্ক্রিপ্টের **সম্পূর্ণ পথ** (Absolute path) দিন।

❌ **ভুল:** `30 6 * * * backup.sh`
✅ **সঠিক:** `30 6 * * * /home/the-king/scripts/backup.sh`

❌ **ভুল:** `0 2 * * * python script.py`
✅ **সঠিক:** `0 2 * * * /usr/bin/python3 /home/the-king/script.py`

### ২. Output Redirection ব্যবহার করুন

Cron job-এর output বা error স্ক্রিনে দেখায় না। সেগুলো ফাইলে সেভ করে রাখুন:

```bash
# Output ও Error একটি ফাইলে সেভ করুন
0 2 * * * /home/user/backup.sh > /home/user/cron-log.txt 2>&1
```

### ৩. Script-কে Executable (`chmod +x`) করুন

Cron যে স্ক্রিপ্ট চালাবে, তার 실행-permisson (Execute permission) থাকতে হবে।

## 5️⃣ `at` কমান্ড — এককালীন কাজ (One-time Task)

**Cron** নিয়মিত কাজের জন্য। কিন্তু কোনো কাজ যদি **শুধু একবার** করতে চান, তবে `at` ব্যবহার করুন।

### ইন্সটল ও চালু করা:

```bash
sudo apt install -y at
sudo systemctl enable --now atd
```

### ব্যবহার:

```bash
# বিকেল ৪টায় কাজ সেট করা
at 4:00 PM
at> sudo apt update -y
at> <Ctrl+D চেপে সেভ করে বের হোন>
# warning: commands will be executed using /bin/sh
# job 1 at Thu Mar  5 16:00:00 2026

# ১০ মিনিট পর
at now + 10 minutes

# আগামীকাল দুপুর ১২টায়
at 12:00 PM tomorrow
```

### `at` Job নিয়ন্ত্রণ:

```bash
# সব at job-এর তালিকা দেখুন
atq
# 1    Thu Mar  5 16:00:00 2026 a the-king

# কোনো job মুছে ফেলুন (ধরি Job ID = 1)
atrm 1
```

## 📊 Cron Syntax Cheat Sheet

```
* * * * * কমান্ড
| | | | |
| | | | +-- সপ্তাহের দিন (0-7, 0 ও 7 উভয়ই রবিবার)
| | | +---- মাস (1-12)
| | +------ মাসের দিন (1-31)
| +-------- ঘন্টা (0-23)
+---------- মিনিট (0-59)
```

**সাধারণ উদাহরণ:**

- `*/15 * * * *` = প্রতি ১৫ মিনিট পর
- `0 0 * * *` = প্রতিদিন মধ্যরাতে
- `0 8 * * 1-5` = সোম থেকে শুক্র সকাল ৮টায়
