## 🧭 Troubleshooting Mindset (কীভাবে ভাববেন?)

লিনাক্স সিস্টেমে সমস্যা হওয়াটা স্বাভাবিক। কিন্তু সমস্যা সমাধানে একজন সাধারণ ইউজার এবং একজন এক্সপার্টের মধ্যে পার্থক্য হলো তাদের **অ্যাপ্রোচ (Approach)**।

1. **প্যানিক (Panic) করবেন না:** লিনাক্সে সবকিছু লজিক্যালি কাজ করে। এররের পেছনে অবশ্যই কোনো কারণ আছে।
2. **লগ (Log) হলো আপনার প্রধান উৎস:** লগ না দেখে আন্দাজে কোনো কমান্ড চালাবেন না (Section 20 মনে আছে?)।
3. **একটি করে পরিবর্তন করুন:** একসাথে অনেক কিছু পরিবর্তন করলে বুঝবেন না কোনটিতে কাজ হলো আর কোনটিতে অবস্থা আরও খারাপ হলো।
4. **"It works on my machine" সিনড্রোম:** আগেরবার কাজ করেছে মানে এবারও করবে, এমন নয়। কনফিগারেশনে কিছু হয়তো বদলেছে।
5. **সবচেয়ে সহজ কারণ দিয়ে শুরু করুন:** ইন্টারনেট নেই? আগে দেখুন ক্যাবল ঠিক আছে কিনা, এরপর রাউটার চেক করুন, শেষে কমান্ড দিন (ওকাম'স রেজর তত্ত্ব)।

## 🔍 ধাপে ধাপে সমস্যা খোঁজা (The 5-Step Process)

ধরি, আপনার **ওয়েব সার্ভার (Nginx/Apache) কাজ করছে না**।

### স্টেপ ১: সার্ভিস কি চলছে (Running)?

প্রথমেই দেখুন সার্ভিসটি ক্র্যাশ করেছে কিনা বা স্টপ হয়ে আছে কিনা।

```bash
sudo systemctl status nginx
```

যদি `active (running)` দেখায়, তাহলে সার্ভিস ঠিক আছে। যদি `failed` দেখায়, তাহলে লগে এরর খুঁজুন।

### স্টেপ ২: রিসোর্স কি ফুরিয়ে গেছে?

সার্ভিস চলতে গেলে RAM বা DISK প্রয়োজন।

```bash
# ডিস্ক ফুল হয়ে গেলে কোনো সার্ভিস কাজ করবে না (No space left on device)
df -h

# RAM ফুল হয়ে গেলে লিনাক্স সার্ভিস কিল করে (OOM Killer)
free -m
```

### স্টেপ ৩: নেটওয়ার্ক ও পোর্ট কি ঠিক আছে?

সার্ভিস چلকানো সত্ত্বেও অনেক সময় সাইট লোড হয় না। এর মানে পোর্ট হয়তো খোলা হয়নি।

```bash
# ওয়েব সার্ভারের 80 এবং 443 পোর্ট খোলা আছে কিনা দেখুন
sudo ss -tulnp | grep nginx
```

### স্টেপ ৪: ফায়ারওয়াল কি ট্রাফিক ব্লক করছে?

পোর্ট খোলা থাকলেও ফায়ারওয়াল (UFW) যদি অনুমতি না দেয়, তবে বাইরে থেকে কেউ সাইট দেখতে পারবে না।

```bash
sudo ufw status
# 80/tcp এবং 443/tcp 'ALLOW' আছে কিনা চেক করুন
```

### স্টেপ ৫: বিস্তারিত লগ ফাইল চেক করা

উপরের সবকিছু ঠিক থাকার পরও যদি কাজ না করে, তবে লগ ফাইল চেক করা অপরিহার্য।

```bash
sudo tail -f /var/log/nginx/error.log
# অথবা
journalctl -u nginx -xe
```

## 🌐 Google এবং StackOverflow-এ সঠিকভাবে খোঁজ করা

"My Linux is not working" লিখে গুগলে সার্চ করলে কোনো সমাধান পাবেন না। সঠিক কিওয়ার্ড ব্যবহার করা জরুরি।

### সার্চ করার ফর্মুলা (Formula):

`[OS Name/Version] + [Software/Command] + [Error Message]`

**❌ ভুল সার্চ:** `apache web server not starting`
**✅ সঠিক সার্চ:** `Ubuntu 24.04 apache2 failed to start AH00558`

### সার্চের কিছু টিপস:

1. লগের মেসেজ হুবহু কপি করে গুগলে কোটেশন மாር্কের `""` ভেতর খুঁজুন।
2. যদি আপনার নিজের ফোল্ডারের নাম লগে থাকে (যেমন: `/home/the-king/file.txt not found`), তবে গুগলে শুধু `file not found` খুঁজুন (কারন আপনার ফোল্ডারের নাম অন্য কারো সাথে মিলবে না)।
3. উত্তরের তারিখ খেয়াল করুন! ২০১৫ সালের ফায়ারওয়ালের কমান্ড ২০২৬ সালের Ubuntu 24.04-এ কাজ নাও করতে পারে।

## 🆘 ফোরাম/কমিউনিটিতে সাহায্য চাওয়া (How to Ask)

StackOverflow, Reddit (r/linuxadmin), বা ফেসবুক গ্রুপে সাহায্য চাইলে এমনভাবে প্রশ্ন করুন যেন অন্যরা আপনাকে সাহায্য করতে বিরক্ত না হয়।

### প্রশ্নের গঠন কেমন হবে:

1. **আমি কী করতে চাইছিলাম?** (What is the goal?)
2. **আমি কী কমান্ড দিয়েছিলাম?** (What did I run?)
   _(টিপস: `history` কমান্ড দিয়ে সঠিক কমান্ডটি কপি করে দিন)_
3. **আমি কী আউটপুট/এরর পেয়েছি?** (What is the error?)
   _(কখনোই স্ক্রিনের ছবি বা মোবাইল দিয়ে তোলা ছবি দেবেন না, লগ বা এরর হুবহু কপি-পেস্ট করে কোড ব্লকে দিন)_
4. **আমি নিজে কী কী সমাধান চেষ্টা করেছি?** (What have I tried so far?)

> 💡 **নিয়ম:** "সাহায্য করুন, কাজ করছে না!" - এমন শিরোনামের চেয়ে "Ubuntu 24-এ Nginx স্টার্ট করার সময় Permission Denied Error" লেখা অনেক বেশি কার্যকরী।

## 🛠️ ৫টি কমন এরর ও প্রথম সমাধান

| এরর মেসেজ                  | সম্ভাব্য কারণ                                                 | প্রথম করণীয়                                                                            |
| -------------------------- | ------------------------------------------------------------- | --------------------------------------------------------------------------------------- |
| `Permission denied`        | আপনার ওই ফাইল পড়ার/রান করার অনুমতি নেই বা আপনি রুট ইউজার নন। | কমান্ডের আগে `sudo` দিন অথবা `ls -l` দিয়ে ফাইলের পারমিশন চেক করুন।                     |
| `Command not found`        | সফটওয়্যারটি ইন্সটল করা নেই অথবা ফাইলটি `PATH`-এ নেই।         | সফটওয়্যারটি `apt` দিয়ে ইন্সটল করুন অথবা সম্পূর্ণ পাথ (Absolute Path) লিখে কাজ সারুন।  |
| `No space left on device`  | হার্ডডিস্ক ফুল হয়ে গেছে। কোনো সফটওয়্যার ইন্সটল হবে না।      | `df -h` চেক করুন এবং `/var/log` ফাইলের বড় লগ মুছে দিন বা পুরোনো জিপ ফাইল সরান।         |
| `Connection refused`       | সার্ভিস চলছে না, ফায়ারওয়াল বাধা দিচ্ছে, অথবা পোর্ট ভুল।     | `systemctl status` দেখুন, `ss -tulnp` দিয়ে পোর্ট চেক করুন এবং UFW রুলগুলো একবার দেখুন। |
| `Unable to locate package` | প্যাকেজের নাম ভুল অথবা রিপোজিটরি আপডেট করা নেই।               | `sudo apt update` দিন এবং প্যাকেজের সঠিক নাম গুগলে সার্চ করুন।                          |

## 🏋️ Practice Exercise

### অনুশীলন ১: সিস্টেম ট্রাবলশুটিং (রিসোর্স চেক)

যখনই সিস্টেম স্লো মনে হবে, এই সিকোয়েন্সটি রান করাবেন:

```bash
# 1. ডিস্ক স্পেস চেক করুন
df -h

# 2. RAM চেক করুন
free -m

# 3. কোন প্রসেস বেশি CPU খাচ্ছে তা দেখুন
htop
# (htop থেকে বের হতে F10 বা q চাপুন)
```

### অনুশীলন ২: কমান্ড খোঁজা এবং লগ দেখা

ধরি আপনি ভুলে গেছেন ফাইল খোঁজার কমান্ড কী:

```bash
# 1. ম্যানুয়াল খোঁজা
man -k "search files"
# আউটপুটে find, locate ইত্যাদি আসবে

# 2. সিস্টেমের লাস্ট ২০টি এরর খুঁজে বের করুন
journalctl -p err -b | tail -n 20
```

## 🎉 Phase 6 এবং সম্পূর্ণ কোর্স Complete! 🏆

অভিনন্দন! আপনি **Ubuntu 24 LTS - Beginner to Pro** গাইডের সবকটি Phase সফলভাবে শেষ করেছেন! 🎊

### এই Phase-এ যা শিখলেন:

- ✅ **Customization:** Environment Variables, `~/.bashrc`, Alias এবং Path।
- ✅ **Archive/Compression:** `tar`, `gzip`, `zip` দিয়ে ফাইল প্যাকেজ ও ব্যাকআপ।
- ✅ **Backup/Restore:** `rsync` দিয়ে 3-2-1 Rule মেনে লোকাল ও রিমোট সিঙ্ক এবং Cron অটোমেশন।
- ✅ **Troubleshooting:** প্যানিক না করে স্টেপ-বাই-স্টেপ প্রবলেম সলভিং এবং হেল্প চাওয়ার নিয়ম।

### আপনার পরবর্তী ধাপ কী? 🚀

1. প্রতিটি সেকশনের কমান্ডগুলো আপনার লিনাক্স মেশিনে বারবার প্র্যাকটিস করুন।
2. নিজের জন্য ছোট ছোট **Shell Script** লিখে দৈনন্দিন কাজ (যেমন আপডেট, লগ ক্লিয়ারিং) অটোমেট করুন।
3. লিনাক্স সিস্টেমে একটি ওয়েবসাইট (যেমন WordPress) **হোস্ট (Host)** করে লাইভ প্রজেক্ট তৈরি করুন।
